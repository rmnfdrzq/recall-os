# Data Model

RecallOS uses Django models backed by PostgreSQL. Vector embeddings use `pgvector.django.VectorField`.

## Entity Overview

```text
Document
  └── DocumentChunk

ChatSession
  └── ChatMessage
```

## Document

Source: `backend/core/models.py`

Purpose: stores uploaded file metadata and AI-generated catalog metadata.

Fields:

| Field | Type | Notes |
| --- | --- | --- |
| `id` | UUID | Primary key |
| `file` | FileField | Stored under `backend/media/documents/` |
| `filename` | CharField | Original uploaded file name |
| `file_type` | CharField | `pdf`, `text`, `markdown`, `image` |
| `status` | CharField | `pending`, `processing`, `processed`, `failed` |
| `suggested_title` | CharField | Generated by metadata extraction |
| `summary` | TextField | Generated summary |
| `category` | CharField | Generated category |
| `tags` | JSONField | Generated tag list |
| `created_at` | DateTime | Auto-created |
| `updated_at` | DateTime | Auto-updated |

Ordering:

```text
-created_at
```

## DocumentChunk

Purpose: stores searchable text slices and vector embeddings.

Fields:

| Field | Type | Notes |
| --- | --- | --- |
| `document` | FK Document | Cascades on document deletion |
| `content` | TextField | Chunk text |
| `chunk_index` | Integer | Ordered position in source document |
| `embedding` | VectorField(768) | Matches `nomic-embed-text` dimensions |
| `created_at` | DateTime | Auto-created |

Ordering:

```text
chunk_index
```

Search uses:

```python
CosineDistance('embedding', query_vector)
```

## ChatSession

Purpose: groups related chat messages in the local workspace.

Fields:

| Field | Type | Notes |
| --- | --- | --- |
| `id` | UUID | Primary key |
| `title` | CharField | Defaults to `New Chat Session` |
| `created_at` | DateTime | Auto-created |
| `updated_at` | DateTime | Auto-updated |

Ordering:

```text
-updated_at
```

## ChatMessage

Purpose: stores user and assistant messages.

Fields:

| Field | Type | Notes |
| --- | --- | --- |
| `session` | FK ChatSession | Cascades on session deletion |
| `role` | CharField | `user` or `assistant` |
| `content` | TextField | Message body |
| `sources` | JSONField | Source chunk references for assistant responses |
| `created_at` | DateTime | Auto-created |

Source reference shape:

```json
{
  "document_id": "uuid",
  "filename": "document.md",
  "suggested_title": "Readable title",
  "chunk_index": 0,
  "snippet": "First 150 characters..."
}
```

Ordering:

```text
created_at
```

## Local Workspace Rules

- The application does not model users or ownership.
- Document queries return local workspace documents.
- Search queries inspect chunks from local workspace documents.
- Chat sessions and messages are shared within the local workspace database.
- The `ChatMessage.role` value `user` means "human chat message", not an application account.

## Migration Notes

The first migration creates all core tables. If embedding dimensions change, a migration and reindexing strategy are required because existing vector rows must match the new dimension.
